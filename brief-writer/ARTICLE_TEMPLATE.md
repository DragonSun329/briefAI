# Article Template

文章结构参考，不是死板模板。

---

## Structure

### 1. HOOK (1-2 sentences)
立刻抓住注意力。不解释，直接给冲击。

Examples:
- 一个反直觉的事实
- 一个个人经历片段
- 一句挑衅性的判断

### 2. SETUP (2-3 sentences)
交代背景，但带观点。

不是："Meta发布了新模型..."
而是："当Meta在凌晨悄悄发了这个，大多数人没意识到发生了什么..."

### 3. THESIS (1-2 sentences)
你的核心观点，直接亮出来。

不藏着掖着。读者到这里应该知道你要说什么了。

### 4. ARGUMENT (主体)
展开论证。分2-3个小点。

每个点：
- 一个claim
- 支撑的evidence
- "这意味着..."

### 5. OBJECTION HANDLING (optional)
预判反对意见，主动回应。

"你可能会说...但是..."

### 6. SO WHAT (2-3 sentences)
对读者的意义。

- 如果你是开发者，这意味着...
- 如果你在创业，你应该...
- 如果你在关注这个领域，接下来看...

### 7. CLOSER (1-2 sentences)
收尾。要有力。

好的收尾：
- 一个预测
- 一个问题
- 一个行动召唤
- 一句金句

## Word Count Targets

| Section | Chinese Characters |
|---------|-------------------|
| Hook | 50-100 |
| Setup | 100-150 |
| Thesis | 50-100 |
| Argument | 800-1200 |
| Objection | 100-200 (optional) |
| So What | 100-200 |
| Closer | 50-100 |
| **Total** | **1500-2200** |

---

## Title Formula

好标题的模式：

1. **数字 + 判断**: "90%的AI创业公司，做错了这一件事"
2. **反直觉**: "为什么我不再用ChatGPT"
3. **预测**: "Agent的下一个战场，不在ToC"
4. **提问**: "OpenAI最大的敌人是谁？"
5. **场景**: "凌晨三点，我想明白了一件事"

---

## 备用结构（叙事型）

对于适合讲故事的话题：

1. Scene（场景切入）
2. Tension（矛盾/问题）
3. Revelation（洞察/转折）
4. Implication（意义）
5. Call（行动/思考）

---

不要死套模板。让内容决定结构。
